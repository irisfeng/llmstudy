# LLM Study 免费推广执行计划

## 核心判断

网站现阶段不缺课程数量，缺的是可信的首批分发和学习激活。推广的最小闭环是：一篇真正解决问题的内容 → 一个对应课程深链 → `campaign_landing` → `lesson_started` → `lesson_completed` / `content_shared`。

每周预算控制在 2–3 小时；不付费投流、不买粉、不批量群发、不把同一广告文案复制到所有社区。

## 优先级

1. GitHub 项目页：长期可信入口，README 直接展示课程价值、截图和三节试听课。
2. V2EX 分享发现：用 build-in-public 方式征求课程结构反馈，不写成“求 Star”广告。
3. 掘金技术长文：每篇只讲透一个问题，正文完整，课程链接放在结尾作为实验延伸。
4. 知乎高意图问答：回答已经存在的具体问题，例如 QKV、BPE、KV Cache；先给完整答案再链接。
5. B站 / 视频号：复用一分钟介绍视频，并把 10 节 GEO 直接答案拆成 30–60 秒技术短视频。
6. 微信群 / 朋友圈：只发给确实关注 AI 学习的人，说明这是个人开源项目并主动征求纠错。

V2EX 的“分享发现”节点明确欢迎“好玩、有信息量”的分享；GitHub 也支持为仓库设置社交预览图。发布时必须保持项目介绍真实、具体、有技术信息，避免只剩推广口号。

## 30 天节奏

| 周 | 发布 | 复用内容 | 目标 |
| --- | --- | --- | --- |
| 第 1 周 | GitHub README + 1 篇 V2EX | 项目为什么做、69 节路线、三节试听 | 首批开发者反馈 |
| 第 2 周 | 1 篇掘金 | 反向传播逆拓扑与 `+=` | 高意图课程访问 |
| 第 3 周 | 1 个知乎回答 | QKV、缩放与 causal mask | 长尾搜索与收藏 |
| 第 4 周 | 1 条 B站 + 1 条视频号 | KV Cache 公式或一分钟课程介绍 | 验证短视频转化 |

同一周只发布一个主内容，其他渠道只做适配后的摘要。发布 24 小时内认真回复所有具体问题；争议和纠错优先沉淀回课程与 GitHub Issue。

## KPI 与决策门槛

- 北极星指标：来自推广渠道的 `lesson_started`，不是曝光量或点赞。
- 首月健康目标：100 个可归因访问、30 个课程开始、10 个课程完成、5 次主动分享、2 个真实外部引用或链接。
- 激活率：`lesson_started / campaign_landing`，目标不低于 25%。
- 内容有效：单篇带来至少 15 个可归因访问，或至少 3 条可执行反馈。
- 连续两次低于门槛：暂停该渠道，先改选题与开头，不增加发布频率。
- 单篇达到 30 个访问且激活率超过 30%：围绕同一主题继续做系列内容。

## 每周复盘

1. Vercel Analytics 查看 `campaign_landing`，按 `source / content` 汇总。
2. 对照 `lesson_started`、`lesson_completed` 和 `content_shared`。
3. 只记录一个本周结论：哪个主题吸引了真正学习者？
4. 下一周只改变一个变量：标题、开头、渠道或目标课程，避免同时改完无法判断。

发布文案和 UTM 深链见 `promotion/ready-to-post.md`；渠道配置由 `npm run qa:promotion` 自动检查。
