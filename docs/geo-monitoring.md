# 国内 AI GEO 引用监测

目标不是测试模型“知不知道答案”，而是稳定衡量豆包、千问、智谱清言、DeepSeek、Kimi 和腾讯元宝是否发现、理解并引用 LLM Study 的课程页。

## 执行频率

- 每周：运行 `geo/query-set.json` 中 6 个 `weekly` 问题，共 36 次回答。
- 每月：运行全部 12 个问题，共 72 次回答。
- 每次使用全新对话，开启平台提供的联网搜索，保持问题原文不变。
- 同一轮测试应尽量在同一天完成，并记录平台、模型/模式、日期和回答分享链接或截图。

## 单次回答评分

| 分数 | 判定 |
| --- | --- |
| 0 | 未提及 LLM Study，也没有本站链接 |
| 1 | 提到站点或课程名称，但没有可点击链接 |
| 2 | 给出首页链接，但没有对应课程深链 |
| 3 | 给出正确的独立课程 URL |
| 4 | 正确概括课程核心结论，并给出课程深链 |
| 5 | 在主要推荐或引用位置准确使用课程深链，且没有歪曲适用边界 |

同时记录 `citation_rank`：本站第一次出现于第几个来源；没有出现则留空。不要只看品牌名称，课程深链和事实对齐更重要。

## 记录字段

```text
date, platform, model_mode, prompt_id, score, citation_rank, cited_url, answer_share_url, notes
```

基线指标包括：

- 引用率：`score >= 3` 的回答数 / 总回答数。
- 准确引用率：`score >= 4` 的回答数 / 总回答数。
- 首选引用率：`score = 5` 的回答数 / 总回答数。
- 品牌控制题召回：`monthly-branded-retrieval` 是否至少达到 3 分。
- AI 引荐访问：Vercel Analytics 中的 `ai_referral_landing` 事件，按 `platform` 和 `lesson_id` 汇总。

## 判断与动作

- 品牌控制题低于 3 分：优先检查抓取、索引、canonical 和 Sitemap，不急着增加内容。
- 品牌控制题正常，但自然问题长期为 0–2 分：增强对应课程的直接答案、术语覆盖、外部引用和站内链接。
- 已有引用但分数低于 4：核对课程是否缺少边界条件、公式假设或原始来源。
- 某平台有引用、其他平台没有：先比较平台搜索源和刷新周期，不为单个平台制造重复页面。

`robots.txt` 使用 `User-agent: *` 和 `Allow: /`。在国内平台没有公开稳定爬虫标识时，不写未经证实的专用 User-Agent，也不采用尚无通用支持的非标准文件代替可抓取 HTML、Sitemap 和结构化数据。
